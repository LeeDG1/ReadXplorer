General Viewer Navigation

First of all: All viewers are synchronized across all opened tracks, for easy navigation and comparison of data. Each viewer allows to jump to any position, by drawing the horizontal scrollbar on top of each viewer. Besides that, panning can be used to scroll through the vicinity of the current position.
Each viewer, which also has a slider beneath the horizontal scrollbar also allows zooming in and out. Another way of zooming is to move the mouse wheel while hovering the viewer. In this case, when zoomed in, the view centers around the currently hovered position.
Each viewer also always allows copying the current mouse position via right click.


About Reference Viewer

The reference viewer, which has to be opened before a track can be viewed, visualizes the reference sequence (genome, chromosome...) with its features (if they are available -> fasta files don't inculde features) in all 6 reading frames.
The features can be selected via left click and their additional menu can be opened via right click. If an features was clicked it can be added to the thumbnail view, and it's sequence can be copied or stored.
The legend panel in the top left corner allows to browse the supported feature types and also allows to show or hide each feature type.


About Track Viewer

The track viewer displays the coverage of it's reference sequence in the current dataset. The coverage is divided into three different classes:
1. Perfect coverage: Depicts all mapped reads that perfectly fit to the reference at their mapping position (0 sequence deviations)
2. Best match coverage: Depicts all mapped reads, that fit to this mapping position best (with least possible sequence deviation rate)
3. Common Coverage: Depicts all mapped reads, that still fit to this mapping position with the maximal allowed error rate during the mapping process, but there is another position in the reference, where they fit with less sequence deviations.
The legend panel in the top left corner allows to browse the coloring of these three classes and also enables the you to show or hide each class.
The options panel next to the legend panel contains two functionalities currently:
1. Enable or Disable automatic scaling of the coverage. If enabled, the coverage will never exceed the visible area of the viewer
2. Coverage normalization either by log2 or by a factor of your own choice (between 0 and 10)


About Track Statistics

  1. Mappings: The total number of mappings in the whole data set, which were mapped to this reference.
  2. Unique mappings: The total number of mappings among the number of "Mappings", which where only mapped once to the reference.
  3. Best match mappings: The total number of mappings belonging to the best match class in the whole track data set.
  4. Perfect mappings: The total number of mappings belonging to the perfect match class in the whole track data set.
  5. Perfect coverage: The percentage of the reference sequence, covered with at least one mapping of the perfect match mapping class.
  6. Best match coverage: The percentage of the reference sequence, covered with at least one mapping of the best match mapping class.
  7. Common coverage: The percentage of the reference sequence, covered with at least one mapping of the common match mapping class.

If the currently viewed track is a sequence pair (paired-end or mate pair) data set, the sequence pair statistic is displayed:

  1. Sequence Pairs: The total number of sequence pairs successfully (= at least one read of the pair) mapped to the reference sequence in the whole track data set.
  2. Perfect sequence pairs: The number of perfect sequence pairs, whose distance is within a given range and whose orientation is as expected.
  3. Smaller sequence pairs: The number of sequence pairs whose distance is smaller than the minimum allowed distance of a perfect pair.
  4. Larger sequence pairs: The number of sequence pairs whose distance is larger than the maximum allowed distance of a perfect pair.
  5. Single mappings: The number of mappings whose partner either did not map to the reference at all, or is already contained in a perfect or smaller distance pair with the read of this mapping and its partner. The latter can occur, when e.g. one read of a sequence pair maps to two or more positions in the reference, but in one position it can form a perfect or smaller distance pair with its partner. Then all other mappings of the multi mapped read are classified as single mappings.
