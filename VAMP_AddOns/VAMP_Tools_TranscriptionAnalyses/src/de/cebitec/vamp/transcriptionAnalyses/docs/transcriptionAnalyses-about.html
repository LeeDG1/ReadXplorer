About Transcription Analyses

The transcription analyses module enables you to carry out different automated analyses for transcriptome data sets.

1. The Transcription Start Site (TSS) detection (enabled by checking the "TSS detection" box) has two default parameters.
A) The minimal coverage increase (absolute read counts) from one position to the neighboring one (default = 50 reads).
B) The minimal coverage increase in percent from one position to the neighboring one (default = 50%).

Additionally, two more parameters can be set:
C) The maximal initial read count (default = 8 reads) can be set
D) The minimal coverage increase (default = 23%) for predicted TSS with less than x (in the default case 8) reads at the position directly in front of the TSS has to be set. It can be the same as the mandatory first parameter, but it is recommended to choose a lower value, depending on your analysis and data quality.

If one of these two additional paramters is > 0 the other one also has to be set correctly.
By checking the Automatic parameter estimation box, the parameters are estimated from the data in the track. In this case the additional parameters are currently not set, only the mandatory parameters.

2. The Filter Genes Analysis allows to filter annotated genes of the reference sequence according to their raw read count. Therefore, it requires
A) The minimum number of reads (default = 100 reads)
B) The maximum number of reads (default = 100000 reads)
mapped within the bounds of an annotation.

3. The Operon detection allows to predict operons for prokaryotic genomes on the basis of:
A) Minimal number of spanning reads (default = 5 reads)
between two annotated genes on the same strand. Reads are called "spanning", if they overlap both neighboring annotations.
For all these analyses the results are returned in a matrix. Furthermore, they are visualized in the corresponding track viewer by different colors. This helps to compare the different analysis results at a glance.
